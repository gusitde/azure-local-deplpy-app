Step 8: Security & Pre-Add Checks

Security profile must match the existing cluster. The pipeline also runs pre-add validations automatically.

{# ── Security Profile ────────────────────────── #}
Security Profile
This must match the existing cluster's security profile.
{# ── Automated Pre-Add Checks ─────────────────── #}
Automated Pre-Add Validations

The pipeline will automatically validate:

  • OS version match – New node must run the same OS build as the cluster
  • Arc registration parity – Same subscription, resource group, region, tenant
  • Azure role assignments – HCI Device Management, Key Vault Secrets User
  • Reserved IP ranges – No overlap with Kubernetes CIDRs (10.96.0.0/12, 10.244.0.0/16)
  • DNS resolution – AD domain, Azure AD, Azure Resource Manager
